# 学习进度追踪表设计文档

> **日期**: 2026-04-10  
> **项目**: Java后端学习进度追踪系统  
> **适用对象**: 前端工程师转Java后端

---

## 1. 项目背景

本项目是一个为前端工程师（有NestJS经验）设计的16周Java后端学习计划。学习资料以markdown文件形式组织，包含详细周计划和补充指南。本追踪系统旨在提供可视化的学习进度管理，增强学习动力和规划能力。

## 2. 设计目标

- **可视化进度展示**：直观展示16周学习进度和总体完成情况
- **记录与对比**：记录实际学习时长，与计划时长对比
- **检查点追踪**：逐条追踪每周学习目标完成情况
- **低维护成本**：自动解析现有markdown内容，无需重复维护数据

## 3. 技术方案

### 3.1 技术栈选择

| 层级 | 技术 | 理由 |
|------|------|------|
| 结构 | 原生HTML5 | 无需构建工具，双击即用 |
| 样式 | 原生CSS | 轻量，无框架依赖 |
| 逻辑 | 原生ES6+ | 现代浏览器完整支持 |
| 构建 | Node.js脚本 | 解析markdown，生成数据 |
| 存储 | LocalStorage | 进度数据本地持久化 |

### 3.2 文件结构

```
tacker/
├── index.html              # 入口页面
├── css/
│   └── main.css            # 全局样式
├── js/
│   ├── app.js              # 应用初始化与事件绑定
│   ├── components.js       # UI组件渲染逻辑
│   ├── storage.js          # LocalStorage CRUD操作
│   └── data.js             # 【自动生成】学习数据
├── scripts/
│   └── build-data.js       # Node构建脚本
└── data/
    └── progress.json       # 【动态生成】用户进度数据
```

### 6.1 核心功能列表

| 数据导出 | 导出进度为JSON文件 | P2 |
| 数据导入 | 从JSON恢复进度 | P2 |

### 6.2 构建脚本规格

**输入**: `week*-detailed-plan.md` 文件
**输出**: `tracker/js/data.js`

**解析规则：**
1. 文件名匹配 `week(\d+)-detailed-plan.md` 提取周号
2. 标题行 `# Week X 详细学习计划: 标题内容` 提取标题
3. 学习目标节 (`## 学习目标`) 提取目标列表
4. 检查清单 (`- [ ] xxx`) 提取检查点
5. 时间分配表中的 `投入时间` 提取预计时长

**错误处理：**
- 文件解析失败时输出警告，跳过该文件
- 缺少可选字段时使用默认值（空数组或0）
- 生成后校验数据完整性

## 7. 界面交互流程

```
用户首次使用:
  1. 运行 `node tracker/scripts/build-data.js` 生成数据
  2. 打开 `tracker/index.html`
  3. 看到16周卡片，全部显示"待开始"
  4. 点击Week 1 → 右侧面板显示详情
  5. 标记状态为"进行中"，开始勾选检查点
  6. 填写实际用时，添加笔记
  7. 所有检查点完成后，状态自动变为"已完成"
  8. 顶部总进度自动更新

用户日常使用:
  - 打开页面即看到上次进度（localStorage自动加载）
  - 点击任意周卡片快速编辑
  - 在侧边栏查看统计概览

周计划更新后:
  - 重新运行构建脚本
  - 刷新页面，新内容自动加载
  - 用户进度数据保留（按周号关联）
```
